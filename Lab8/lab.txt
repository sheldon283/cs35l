   dynamically linked libraries are:
   access, open, fstat, read, mmap, munmap, mprotect, close

   The syscall open open the location of the libraries on the system. 
   fstat uses the file descriptor we get from open to calculate the size of the file, which
   would allow the program to know how much space to allocate in the process's virtual 
   memory. read is used after opening to make sure the file is valid. mmaps creates a new 
   mapping in the virtual address space for the calling process. munmap deletes the mappings
   for a specified address range and causes further references to addresses within the range
   to generate invalid memory references. This therefore unlinks libraries we don't need 
   anymore after dynamically linking the new libraries. mprotect protects the processes's 
   memory of the libraries. close just closes the library after the code has been 
   dynamically linked.
